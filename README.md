# RL_Ensemble
RL model to predict actions for stock trading


Instructions:
1. “brew install python@3.12”

     Run the following command “python3.12 —version” to see the version installed through    home-brew.


2.  “Cd” to your working directory and clone the repository
          “git clone  https://github.com/abinayg/RL_Ensemble.git “

3. You should see a folder “RL_Ensemble” in your current working directory


4. Setup the python virtual environment
         “python3.12 -m venv <env_name>”

5. Activate the virtual environment
         “ source <path_to_env>/bin/activate “

6. Install the required modules from the requirements.txt
         “pip install -r ~/RL_Ensemble/requirements.txt”

7. Open Visual Code Studio and select this virtual environment kernel.

8. Run the “final_script.ipynb” and if everything is followed correctly, you should be able to see no issues in running the notebook.



Below content(project overview) is generated by Gemini CLI. 


  This project is a Reinforcement Learning (RL) Ensemble system designed to predict
  trading actions (Buy, Sell, Hold) for stock trading. It employs multiple RL
  algorithms and an ensemble strategy to improve prediction robustness.

  Project Overview


   - Core Technology: Python 3.12, Reinforcement Learning.
   - Main Frameworks:
       - Stable Baselines3 (https://stable-baselines3.readthedocs.io/) for RL
         algorithms (DQN, PPO, A2C).
       - Gymnasium (https://gymnasium.farama.org/) for the trading environment.
       - FinRL (https://github.com/AI4Finance-Foundation/FinRL) for financial
         reinforcement learning utilities.
       - yfinance (https://github.com/ranaroussi/yfinance) for fetching market data.
   - Architecture:
       - Custom Environment: MultiAssetTradingEnv.py defines a multi-asset trading
         environment with realistic constraints like transaction costs, slippage,
         and cash reserves.
       - Ensemble Strategy: Combines predictions from multiple models (DQN, PPO,
         A2C) and multiple seeds using majority voting to determine final trading
         decisions.
       - Utility Modules: test_helper.py and dummy.py provide functions for data
         downloading, feature engineering (using technical indicators, VIX,
         turbulence), training, and evaluation.
       - Saved Models: Pre-trained models are stored in directories like
         a2c_model_exp_seed_*, dqn_model_exp_seed_*, and ppo_model_exp_seed_*.

  Building and Running


  Prerequisites
   - Python 3.12 (Installable via Homebrew: brew install python@3.12)
   - Virtual Environment


  Setup
   1. Initialize Virtual Environment:
   1    python3.12 -m venv pyenv
   2    source pyenv/bin/activate
   2. Install Dependencies:
   1    pip install -r RL_Ensemble/requirements.txt
     Note: Includes specialized dependencies like `finrl` (from GitHub) and
  `TA-Lib`.


  Running the Project
   - The primary entry point is the Jupyter Notebook:
     RL_Ensemble/final_script.ipynb.
   - Open this notebook in VS Code or Jupyter Lab, select the pyenv kernel, and run
     the cells.

  Development Conventions


   - Environment Logic:
       - Actions are interpreted as [Buy, Hold, Sell] (Discrete) or via a continuous
         Box space determining weights and confidence levels.
       - Rewards are calculated based on portfolio value changes, penalizing
         drawdowns and unnecessary turnover.
       - Cash reserves are maintained (default 10%, min 5% when confident).
   - Data Preprocessing:
       - Uses YahooDownloader for data retrieval.
       - FeatureEngineer adds technical indicators (MACD, CCI, DX, SMA), VIX, and
         turbulence.
       - VecNormalize is used to normalize observations for the RL models.
   - Testing and Validation:
       - Models are evaluated using backtesting logic in test_helper.py.
       - Visualization tools in MultiAssetTradingEnv.py and test_helper.py generate
         plots of price vs. trades and portfolio value.
   - File Naming:
       - Model folders: {algo}_model_exp_seed_{seed}.
       - Plots: {algo}_{ticker}_for_{date}.png or ensemble_{ticker}_for_{date}.png.


  Key Files
   - RL_Ensemble/MultiAssetTradingEnv.py: Custom Gymnasium environment for trading.
   - RL_Ensemble/test_helper.py: Core utilities for ensemble voting, backtesting,
     and plotting.
   - RL_Ensemble/dummy.py: Additional helper functions for training and parameter
     management.
   - RL_Ensemble/final_script.ipynb: Main execution notebook for training and
     evaluating the ensemble.
   - RL_Ensemble/requirements.txt: Python package dependencies.
